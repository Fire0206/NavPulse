# NavPulse

> 面向基金投资者的实时估值与个人投资组合追踪系统

NavPulse 净脉：一款面向基金投资者的实时估值与组合跟踪 Web 应用。支持盘中动态估值、持仓盈亏全景透视、联接基金底层穿透及多用户账户隔离。同时适配电脑端和移动端，随时直观掌握全市场热度与个人资产波动。

## 核心架构与设计理念

- **盘中动态估值引擎**：告别静态的日终净值，基于底层持仓权重与实时行情动态推演盘中估值变动。
- **底层资产穿透测算**：针对联接基金自动下钻至底层 ETF，规避资产标的错位，确保仓位测算与实际市场共振。
- **多维组合透视联动**：将单只基金走势、投资组合总体盈亏、大盘热度及资金分布情况整合于统一的高信息密度视图中。
- **移动端优先与轻量化**：采用极简的单页应用（SPA）交互设计，兼顾碎片化时间的高频查看体验与完整的独立后端部署能力。

## 特性概览

- **双轨视图管理**：独立运作“自选观察池”与“真实持仓组合”，数据互不干扰。
- **深度基金档案**：聚合历史净值曲线、盘中分时估值图、交易流水标记与底层持仓结构分析。
- **智能 OCR 识别导入**：集成图像识别技术，支持通过持仓截图批量解析并自动化导入账户数据。
- **全景市场监测**：直观展示大盘核心指数、全市场涨跌分布图与板块热力追踪，辅助宏观认知。
- **细粒度隐私与多主题**：内置包含夜间模式在内的多套视觉主题，并提供可动态配置的资金金额脱敏保护。
- **多租户安全体系**：基于 JWT 鉴权的标准化账号系统，实现多用户环境下的绝对数据隔离。
- **清晰的状态反馈**：针对缓存刷新、官方净值校准及后台任务调度，提供全局可视化状态指示。

## 核心功能模块

| 模块 | 能力描述 |
|------|------|
| 实时估值 | 盘中高频测算基金涨跌幅与动态净值 |
| 组合管理 | 全生命周期管理持仓成本、日内收益与历史交易流水 |
| 自选观察 | 独立于实盘持仓的长期跟踪池管理 |
| 行情聚合 | 大盘指数、板块热点、市场涨跌分布与个基龙虎榜概览 |
| 历史回溯 | 历史净值走势分析、关键交易节点复盘与日内分时追踪 |
| 数据持久化 | 基于 SQLite 的多级缓存策略与用户状态本地落盘 |
| 鉴权体系 | 标准化注册登录流程、JWT 状态保持及隔离网关 |
| 任务调度 | 异步定时刷新市场行情、校准估值底座与清理应用缓存 |

## 部署与运维须知

在将服务部署至生产环境（如宝塔面板或自有服务器）前，请严格注意以下数据隔离与安全事项：

- **运行期数据排查**：`app/data/navpulse.db` 及其扩展文件（`-wal`, `-shm`）属于运行时数据库，切勿将本地开发环境的数据库连同代码一并推送到线上环境。
- **安全密钥配置**：生产环境启动前，**必须**配置环境变量 `JWT_SECRET_KEY`，否则服务端鉴权模块将引发异常阻断启动。
- **环境隔离最佳实践**：建议线上环境仅拉取纯净代码，由应用在初次运行时利用自动初始化机制构建干净的数据库实例，避免本地日志与缓存污染生产环境。

## 快速开始

```bash
git clone [https://github.com/Fire0206/NavPulse.git](https://github.com/Fire0206/NavPulse.git)
cd NavPulse
python -m venv venv
source venv/bin/activate  # Windows 环境请使用 venv\Scripts\activate
pip install -r requirements.txt
````

**开发模式启动：**

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

**生产环境启动：**

```bash
gunicorn -c gunicorn_conf.py app.main:app
```

> 注：更详尽的生产级部署方案（含 Nginx 反向代理与进程守护配置）请参阅 [DEPLOY.md](DEPLOY.md)。

## 环境变量配置指南

| 变量 | 说明 |
|------|------|
| `JWT_SECRET_KEY` | JWT 签名密钥（生产环境必填且需保持强复杂度固定值） |
| `ENVIRONMENT` | 运行环境标识（生产环境建议设为 `production`） |
| `CORS_ORIGINS` | 允许跨域访问的前端域名白名单（生产环境严禁使用 `*`） |
| `TOKEN_EXPIRE_MINUTES` | 登录凭证（Token）有效生命周期 |
| `DATABASE_URL` | 数据库连接字符串（默认适配 SQLite） |
| `DEBUG` | 开发调试模式开关 |
| `PORT` | 内部服务监听端口 |

## 技术栈

  - **后端服务层**：FastAPI, SQLAlchemy, APScheduler
  - **数据采集层**：akshare
  - **前端交互层**：Vue 3, Bootstrap 5, ECharts
  - **持久化与网关**：SQLite, Gunicorn, Uvicorn

## 项目定位与适用场景

  - 寻求精准盘中估值参考，不满足于代销平台日终静态净值的基金投资者。
  - 需要整合分散于多个券商/第三方平台的持仓，实现统一集中监控的个人投资者。
  - 金融科技方向的开发者，作为量化工具、定制化行情监控系统的二次开发脚手架与参考实现。
