# NavPulse

> 一个面向基金投资者的实时估值与组合跟踪应用。打开网页，就能看到持仓基金当前大致涨跌、组合盈亏、市场温度和盘中波动，而不是等到晚上才知道今天发生了什么。

NavPulse 的核心目标不是“展示基金资料”，而是把基金持仓、重仓股变化、市场结构和你的个人组合放到同一个界面里，尽量把今天的行情变化更早、更直观地呈现出来。

## 为什么它值得放进你的 GitHub 主页

- 实时估值不是静态数据页，而是基于基金持仓与股票实时行情推导出的盘中估算结果。
- 既能看单只基金，也能直接看整组持仓的今日收益、估算净值、交易轨迹和资金分布。
- 支持联接基金穿透到底层 ETF，尽量避免“看到的是基金，算的却不是真正仓位”的偏差。
- 提供行情总览、板块热度、涨跌分布和状态栏，打开应用先看到市场，再看到自己的仓位。
- 整体体验偏移动端优先，适合手机上随手打开查看，不是只给桌面浏览器准备的后台页面。

## 功能亮点

- 实时基金估值：根据基金披露持仓与实时行情，动态估算盘中涨跌。
- 组合视角展示：同时跟踪总市值、当日收益、累计收益和单基金表现。
- 自选与持仓分离：关注池和真实持仓两套视图独立管理。
- 基金详情页：查看历史净值、盘中分时估值、交易记录与持仓结构。
- 联接基金穿透：识别联接基金，自动下钻到底层 ETF 持仓计算估值。
- OCR 导入：支持从截图识别并批量导入基金持仓。
- 市场总览：展示指数、全市场涨跌分布、板块热度和涨跌榜。
- 状态反馈清晰：缓存刷新、官方净值更新、后台调度状态都有可视化提示。
- 多主题与隐私模式：兼顾视觉风格和公开场景下的持仓保护。
- 多用户登录体系：支持注册、登录、JWT 鉴权和独立账户数据隔离。

## 使用场景

- 想在收盘前大致判断今天基金表现，而不是等官方净值。
- 想把分散在多个 App 里的基金持仓集中到一个页面统一观察。
- 想快速知道“今天到底是市场整体在涨，还是只有我持有的板块在动”。
- 想给自己的基金工具项目做一个更完整、可部署、可演示的 Web 成品。

## 界面与体验特点

- 手机优先的单页布局，适合随手打开。
- 首页信息密度高，但尽量控制在一屏可读。
- 下拉刷新、状态栏反馈、卡片式数据模块让高频查看更顺手。
- 登录、注册、设置、导入等非核心页面也保持统一的视觉风格。

## 核心能力概览

| 模块 | 能力 |
|------|------|
| 实时估值 | 盘中估算基金涨跌与估算净值 |

## 部署前必须知道

如果你是第一次部署到宝塔，最容易踩的坑不是代码本身，而是运行时文件一起被上传：

- `app/data/navpulse.db` 是运行期数据库，不建议跟代码一起上传。
- `app/data/navpulse.db-wal` 和 `app/data/navpulse.db-shm` 也是 SQLite 运行期文件，不应该作为部署包的一部分。
- 生产环境必须设置 `JWT_SECRET_KEY`，否则服务可能无法按预期启动。
- 如果你上传的是本地开发目录，可能会把本地已有账户、缓存、日志一并带到线上。

建议做法：只上传代码文件，不上传 `app/data/` 下的数据库文件，让服务在服务器上自行初始化。

## 快速开始

```bash
git clone https://github.com/Fire0206/NavPulse.git
cd NavPulse
python -m venv venv
source venv/bin/activate  # Windows 请改用 venv\Scripts\activate
pip install -r requirements.txt
```

开发启动：

```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

生产启动：

```bash
gunicorn -c gunicorn_conf.py app.main:app
```

更完整的生产部署步骤见 [DEPLOY.md](DEPLOY.md)。

## 环境变量

| 变量 | 说明 |
|------|------|
| `JWT_SECRET_KEY` | JWT 签名密钥，生产环境必须固定配置 |
| `ENVIRONMENT` | 运行环境，生产建议设置为 `production` |
| `CORS_ORIGINS` | 允许访问的前端域名，生产不要保留 `*` |
| `TOKEN_EXPIRE_MINUTES` | 登录态有效期 |
| `DATABASE_URL` | 数据库连接串，默认使用 SQLite |
| `DEBUG` | 开发模式开关 |
| `PORT` | 服务监听端口 |
